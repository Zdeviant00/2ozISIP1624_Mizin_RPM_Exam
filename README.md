Простые типы — это базовые «кирпичики»: числа (int, float), логические значения (bool), одиночные символы (char). Они хранят одно значение и занимают мало места.
Структурированные — это «контейнеры», которые объединяют несколько значений: массивы (int[]), списки (List<T>), классы (class Student).

Главное отличие: простые типы живут в «быстрой» памяти (стек), а структурированные — в «медленной» (куча), и доступ к ним идёт через ссылку-адрес.

Пример из жизни:

Простой тип: возраст студента (int age = 20).

Структурированный: карточка студента с именем, возрастом и оценками (class Student { string Name; int Age; double GPA; }).

Выбор типа влияет на:

Память: byte (1 байт) vs long (8 байт) — для миллиона чисел разница в 7 МБ.

Скорость: целые числа (int) считаются быстрее дробных (double).

Точность: float и double дают погрешность (0.1 + 0.2 = 0.30000004), а decimal считает точно, но медленнее.

Типичные ошибки:

Переполнение: int (максимум ≈ 2 млрд) * 2 → отрицательное число. Лечится использованием long.

Потеря точности: использование float для денег. Лечится decimal.

Целочисленное деление: 5 / 2 = 2, а не 2.5. Нужно привести одно число к double.

</details><details> <summary><strong>7. Раскройте понятие «область видимости переменной». Как она влияет на работу подпрограмм?</strong></summary>
Область видимости — это часть кода, где переменная существует и доступна.

Локальная (внутри метода): живёт только во время выполнения метода, исчезает после выхода.

Поле класса (объявлена в классе): доступна всем методам этого класса.

Параметр метода: виден только внутри этого метода.

Модульность — это разбиение программы на маленькие методы, каждый из которых решает одну задачу (принцип единственной ответственности).

Код становится читаемым.

Легко тестировать (каждый метод отдельно).

Удобно менять логику (правка в одном месте).

Методы можно переиспользовать в разных частях программы.

// И вызываем его где нужно.
Теперь изменение формата даты делается в одном месте.

Массив — это набор элементов одного типа, расположенных в памяти последовательно. Размер задаётся при создании и не меняется.

Эффективен, когда:

Нужен быстрый доступ по номеру (индексу).

Требуется перебор всех элементов (цикл for).

Пример: хранение оценок студента (int[] scores = {5, 4, 3, 5};).

Ограничение: размер нельзя изменить. Если нужно добавлять элементы — используйте List<T>.

</details><details> <summary><strong>11. Объясните, как работа со строками отличается от работы с другими типами данных. Приведите примеры операций со строками.</strong></summary>
Строки (string) — это неизменяемые (immutable) ссылочные типы. Это значит, что любая операция (замена, обрезка, приведение к верхнему регистру) создаёт новую строку, а не изменяет старую.

Особенности:

Сравниваются по содержимому ("hello" == "hello" — true), а не по ссылке.

Много встроенных методов: ToUpper(), Substring(), Replace(), Split(), Trim().

Если нужно много операций со строками (в цикле), используйте StringBuilder — он изменяемый и не создаёт кучу мусорных объектов.

По значению (по умолчанию): метод получает копию. Изменения внутри метода не видны снаружи.

out: как ref, но переменную не нужно инициализировать до вызова. Метод обязан присвоить ей значение.
Используйте: для возврата нескольких значений (например, int.TryParse).

in: передача по ссылке только для чтения (оптимизация для больших структур).

Наследование позволяет переиспользовать код.

</details><details> <summary><strong>30. Объясните, что такое Binding в WPF. Как работает Binding и какие режимы Binding существуют?</strong></summary>
Binding (привязка) — это автоматическая синхронизация между интерфейсом и данными. Изменение данных → обновление UI, и наоборот.

Режимы:

OneWay — только из данных в UI (например, TextBlock).

TwoWay — в обе стороны (например, TextBox).

OneWayToSource — только из UI в данные.

OneTime — данные читаются один раз при загрузке.

Чтобы Binding работал, в классе-источнике нужно реализовать интерфейс INotifyPropertyChanged — он уведомляет UI об изменениях.

</details><details> <summary><strong>37. Объясните, как выполняются асинхронные операции в C#. Как работают ключевые слова async и await?</strong></summary>
Асинхронность позволяет не блокировать интерфейс во время долгих операций (загрузка файлов, запросы к БД).

Ключевые слова:

async — отмечает метод как асинхронный.

await — ожидает завершение операции, но не блокирует поток.

Пример:

csharp
private async void Button_Click(object sender, RoutedEventArgs e) {
    string data = await DownloadDataAsync();  // UI не зависнет
    MessageBox.Show("Готово!");
}
Пока идёт загрузка, интерфейс остаётся отзывчивым.

Важно: метод, помеченный async, должен возвращать Task (если ничего не возвращает), Task<T> (если возвращает значение) или void (только для обработчиков событий).
