<details> <summary><strong>1. Объясните, чем отличаются простые типы данных от структурированных. Приведите примеры их использования в реальных задачах.</strong></summary>
Простые типы — это базовые «кирпичики»: числа (int, float), логические значения (bool), одиночные символы (char). Они хранят одно значение и занимают мало места.
Структурированные — это «контейнеры», которые объединяют несколько значений: массивы (int[]), списки (List<T>), классы (class Student).

Главное отличие: простые типы живут в «быстрой» памяти (стек), а структурированные — в «медленной» (куча), и доступ к ним идёт через ссылку-адрес.

Пример из жизни:

Простой тип: возраст студента (int age = 20).

Структурированный: карточка студента с именем, возрастом и оценками (class Student { string Name; int Age; double GPA; }).

</details><details> <summary><strong>2. Опишите, как выбор типа данных влияет на эффективность программы. Какие ошибки могут возникнуть при неправильном выборе типа данных?</strong></summary>
Выбор типа влияет на:

Память: byte (1 байт) vs long (8 байт) — для миллиона чисел разница в 7 МБ.

Скорость: целые числа (int) считаются быстрее дробных (double).

Точность: float и double дают погрешность (0.1 + 0.2 = 0.30000004), а decimal считает точно, но медленнее.

Типичные ошибки:

Переполнение: int (максимум ≈ 2 млрд) * 2 → отрицательное число. Лечится использованием long.

Потеря точности: использование float для денег. Лечится decimal.

Целочисленное деление: 5 / 2 = 2, а не 2.5. Нужно привести одно число к double.

Тип	Размер	Точность	Когда использовать
int	4 байта	Точная (целые)	Счётчики, индексы, идентификаторы
float	4 байта	~7 знаков	Графика, игры, где скорость важнее точности

Для денег всегда используйте decimal (16 байт) — он считает в десятичной системе без погрешностей.

</details><details> <summary><strong>4. Какие условные конструкции существуют в C#? Объясните разницу между ними и приведите примеры ситуаций, когда каждая из них наиболее уместна.</strong></summary>
if-else — для проверки сложных условий (диапазоны, логические И/ИЛИ).
Пример: проверка пароля (длина > 6 И есть цифра).

switch — для сравнения одной переменной с набором конкретных значений. Удобен с перечислениями.
Пример: выбор пункта меню (1 — Добавить, 2 — Удалить).

Тернарный оператор ?: — для простого присваивания в одну строку.
Пример: string status = (age >= 18) ? "Взрослый" : "Ребёнок";

</details><details> <summary><strong>5. Объясните различия между циклами с постусловием, предусловием и параметром. В каких ситуациях каждый из них наиболее эффективен?</strong></summary>
while (предусловие) — проверка ДО выполнения. Если условие ложно — тело не выполнится ни разу.
Используйте: когда неизвестно число итераций (чтение файла до конца).

do-while (постусловие) — проверка ПОСЛЕ. Тело выполнится минимум 1 раз.
Используйте: для меню, где нужно показать опции хотя бы один раз.

for (параметр) — счётчик, условие и шаг в одной строке.
Используйте: когда точно знаете число повторений (обход массива).

foreach — перебор коллекции без индекса.
Используйте: когда индекс не важен (например, вывод списка имён).

</details><details> <summary><strong>6. Сравните процедуры и функции. В каких случаях следует использовать каждый из этих подходов? Приведите примеры.</strong></summary>
И то, и другое — это методы, но:

Функция возвращает значение (int, string, ...). Используется для вычислений.

Процедура не возвращает значение (void). Используется для действий (вывод на экран, сохранение в файл).

Примеры:

csharp
// Функция
static int Sum(int a, int b) => a + b;

// Процедура
static void PrintHello(string name) => Console.WriteLine($"Привет, {name}!");
